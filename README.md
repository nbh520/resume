   我在项目中的角色：主要开发者

   项目责任描述：

   使用短信注册、获取短信验证码
   添加购物车、搜索组件、评价列表
   路由缓存，优化压缩大小
   
   碰到的问题：商品加入购物车后有勾选开关和商品数量以及商品总价，需要将功能全串联在一起，数据同步，并且商品可加入购物车最大数量===该商品库存量，如果使用获取该id以及再次调取接口获取商品信息，将会多很多请求，影响项目加载。

  解决方式：使用Vuex，响应式储存状态，state储存car，接着从getters调取id以及数据因此方便快捷的解决了问题。
